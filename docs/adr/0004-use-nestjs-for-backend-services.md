# ADR-0004: Use NestJS For Backend Services

## Status

Accepted

## Context

Backend services need a TypeScript framework that supports REST APIs, dependency injection, modular structure, validation, testing, and integration with messaging and persistence.

## Decision

ECMP backend services will use NestJS.

## Consequences

NestJS provides a consistent backend structure across services and fits well with TypeScript, clean architecture, and test-driven development. The project accepts NestJS framework conventions and dependency injection patterns.
